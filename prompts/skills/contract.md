# スキル: 契約書草案作成

## このスキルのゴール

法的に必要な条項を網羅した契約書草案を作成する。
後のトラブルを防ぐために、曖昧な表現を排除し、両当事者の権利義務を明確化すること。

## 必ず含める条項（契約種別による）

### 全契約共通
1. 前文（当事者名・契約目的）
2. 定義（重要用語の定義）
3. 契約期間・効力発生日
4. 対価・支払条件
5. 秘密保持義務
6. 禁止事項
7. 契約解除条件
8. 損害賠償・免責
9. 準拠法・管轄裁判所
10. 協議条項

### 業務委託契約に追加
- 業務内容・成果物の定義
- 納期・検収方法
- 再委託の可否
- 知的財産権の帰属
- 独立事業者の地位（雇用関係の否定）

### NDA（秘密保持契約）に追加
- 秘密情報の定義（除外事項含む）
- 情報の管理方法
- 第三者への開示制限
- 情報の返還・廃棄義務
- 有効期間（契約終了後も継続するか）

## 出力フォーマット

```
# 〇〇契約書

甲（依頼者）: 〇〇株式会社（以下「甲」という）
乙（受託者）: 〇〇株式会社（以下「乙」という）

甲乙は、以下のとおり契約を締結する。

---

第1条（目的）
本契約は、〇〇について定めることを目的とする。

第2条（定義）
本契約において、以下の用語は次の意味を有する。
（1）「〇〇」とは、〇〇をいう。

...（各条項）...

---

以上、本契約の成立を証するため、本書2通を作成し、甲乙それぞれ記名押印のうえ、各1通を保有する。

YYYY年MM月DD日

甲: 〇〇株式会社
   代表者: 〇〇

乙: 〇〇株式会社
   代表者: 〇〇
```

## 品質チェックリスト

- [ ] 当事者の定義が明確か（「甲」「乙」の取り違えがないか）
- [ ] 金額・期間・数量が全て具体的な数字で書かれているか
- [ ] 「場合がある」「ことができる」の主体が明確か
- [ ] 解除条項が片務的になっていないか（双方に公平か）

## 入力情報が不足している場合の対応

- **契約の種類のみ（詳細不明）**: 最も一般的なひな型で作成。不明な箇所は「【〇〇を記入】」形式のプレースホルダーを使う
- **金額・期間が未定**: 「〇〇円（税抜）」「202X年〇月〇日〜202X年〇月〇日」と記入欄を設ける
- **当事者の詳細がない**: 「甲（依頼者）: 〇〇株式会社（以下「甲」という）」と様式だけ整えて作成する
- **NDA・業務委託どちらか不明**: 「業務委託契約（NDA条項含む）」として1本化したひな型で作成する

「情報がないので作れません」とは言わない。プレースホルダーを使って完全な骨格を作り、ユーザーが埋めやすい形で提供する。

## ビフォー/アフター例

### ❌ NG出力（このような契約書・チェックは出力しない）

```
第8条（損害賠償）
甲または乙が本契約に違反した場合、相手方に生じた損害を賠償する。

この条項は標準的な内容です。特に問題はないと思われます。
```

**何がダメか**:
- 損害賠償の上限額がない（無限責任状態）
- 「問題はないと思われます」という曖昧な評価
- リスクレベルが明示されていない
- 修正案がない

### ✅ OK出力（このレベルが最低基準）

```
第8条（損害賠償）
1. 甲または乙が本契約に違反し、相手方に損害を与えた場合、違反した当事者は
   相手方に対し損害賠償の責任を負う。
2. 前項に基づく損害賠償額の上限は、本契約に基づき甲が乙に支払った報酬の
   総額を上限とする。
3. 前項の規定にかかわらず、当事者の故意または重大な過失による場合は
   前項の制限を適用しない。

【この条項について: リスクレベル🟡 要注意】
第2項の「報酬総額」の解釈について: 月額契約の場合「支払済みの総額」か
「契約期間全体の総額」かが争われる可能性があります。
「本契約に基づき実際に支払われた報酬の累計額」と明確化することを推奨します。
```

---

## よくある失敗パターンと回避策

1. **損害賠償の上限を設定しない**
   - 上限なしの損害賠償条項は🔴致命的リスク
   - 業務委託なら「報酬総額が上限」が基本。必ず入れる

2. **知的財産権の帰属を書かない**
   - 「何も書かない = 制作物の著作権は受託者（乙）に残る」が原則（著作権法第17条）
   - 「甲に帰属させる」「乙が帰属するが甲に使用許諾する」を明記しないと後でトラブルになる

3. **「業務内容」が曖昧**
   - NG: 「乙は甲の指示に従い、マーケティング業務を行う」
   - OK: 「乙は別紙作業仕様書（本契約の一部）に記載された業務を行う」→仕様書を別紙で添付

4. **契約解除条件が片務的**
   - 甲だけが解除できて乙は解除できない条件、またはその逆は問題
   - 双方に「30日前の書面通知で解除できる」条件を設ける

5. **秘密保持の範囲が広すぎるまたは狭すぎる**
   - 「一切の情報」は広すぎ（日常会話も含まれる）
   - 「書面で秘密と明示したもの」は狭すぎ（口頭情報が守られない）
   - 「書面で秘密指定したもの・口頭開示後5日以内に書面確認したもの」が適切

## 必須チェック項目（出力後の自己確認）

出力後、以下を全て確認する。1つでも「×」があれば書き直す。

1. [ ] 損害賠償条項に上限額が設定されているか
2. [ ] 知的財産権の帰属が明記されているか
3. [ ] 業務範囲・成果物の定義に「別紙仕様書」または具体的な記述があるか
4. [ ] 解除条件が甲乙双方に設けられているか（片務的でないか）
5. [ ] 末尾に弁護士確認の免責事項が記載されているか
