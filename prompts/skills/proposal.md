# スキル: 提案書作成

## このスキルのゴール

A4換算5ページ以内のREQS Lab標準提案書を作る。
WHY→WHAT→HOWの構成で、クライアントの言葉で課題を語る。
「サービス説明書」ではなく「意思決定を促すストーリー」として機能すること。

## 出力構成（必須）

### ページ1：エグゼクティブサマリー
- 課題（1〜2行、クライアントの言葉で）
- REQS Labのソリューション（1〜2行）
- 期待効果（必ず数字で表現）

### ページ2：現状分析と課題定義
- 現状の整理
- 課題の核心（「〜したいが、〜ができていない。なぜなら〜だから」の形式で）
- このまま続けた場合のリスク

### ページ3〜4：ソリューション
- WHY（なぜこの解決策か、3つ以上の論拠）
- WHAT（何をするか）
- HOW（フェーズ別の進め方、最低3ステップ）

### ページ5：費用・スケジュール・ネクストステップ
- 投資対効果（コストより先にリターンを書く）
- 費用プラン（明確に。「〜円〜」の範囲表記でも可）
- スケジュール
- 次のアクション（具体的な期日と行動。「ご検討ください」は絶対NG）

## 業種別トーン指定

- 商店街系 → 親しみやすく・技術用語NG・地域への愛情を見せる
- 医療系 → 誠実・正確・個人情報への配慮を明示
- 採用系 → 熱く・採用担当者に共感・コスト削減の数字を強調
- 自治体系 → 丁寧・長期視点・社会的インパクトを強調
- スタートアップ系 → 数字で語る・スピード感・実績前面

業種が不明な場合は文脈から推測して最適なトーンを選ぶ。

## ブランドルール（必ず守る）

- **断言する**（「〜します」「〜です」。「〜かもしれません」「〜と思われます」はNG）
- 「弊社」「当社」は使わない
- 「ご検討ください」はNG → 「一緒に進めましょう」「まず〇日までに〇〇を」
- A4換算5ページを超えない
- 効果は必ず数字・期間・Before/Afterで具体化（「大幅に改善」等の曖昧表現NG）

## 入力情報が不足している場合の対応

ユーザーからの情報が少ない場合でも、作業を止めずに以下の方針で対応する:

- **クライアント名のみ**: 業種を推測（または括弧書きで仮定を明示）して作成。例: 「〇〇食品様（食品メーカーと仮定）」
- **課題が不明**: 業種に典型的な3つの課題を仮定して提案し、「実際の課題に合わせて修正します」と末尾に記載
- **予算が不明**: 「スモール（〜50万円）」「スタンダード（50〜200万円）」「プレミアム（200万円〜）」の3プランを簡単に提示
- **業界情報がない**: 「一般的な中小企業」を前提として作成し、業界固有の要素はプレースホルダー（【要確認: 業界特有の数字】）で示す

情報が少なくても「作れません」とは言わない。仮定を置いて作成し、仮定箇所を明示する。

## ビフォー/アフター例

### ❌ NG出力（このような提案書は出力しない）

```
## エグゼクティブサマリー

貴社の現状の課題を解決するため、弊社のサービスをご提案申し上げます。
導入いただくことで、業務の効率化が期待できます。
ぜひご検討ください。

## 1. 現状と課題

現在、貴社では様々な課題を抱えていらっしゃるかと思われます。
特に人材不足の問題は、多くの企業にとって深刻な問題ではないでしょうか。
```

**何がダメか**: 
- 数字が一つもない
- 「〜かと思われます」「〜ではないでしょうか」の曖昧表現
- 課題が抽象的すぎてどの会社にも当てはまらない
- 「ご検討ください」でCTA放棄

### ✅ OK出力（このレベルが最低基準）

```
## エグゼクティブサマリー

**課題**: 〇〇社は月40時間を紙の見積書作成に消費。ミス率12%・顧客への送付が平均3日遅延。
**解決策**: 見積書自動生成システムの導入（初期設定2週間、移行コスト10万円）
**効果目安**: 作業時間40時間→8時間（-80%）、ミス率12%→2%以下、送付を当日中に改善

## 1. 現状と課題

**現状**: 見積書作成に月40時間（=月8万円の人件費）が消費されている
**課題の核心**: Excelによる手作業→転記ミス→手直し→送付遅延のループを断ち切れていない
**放置リスク**: 競合他社が見積提出まで半日以内に対応している中、3日遅延は失注の直接原因になる
（実際、昨年度の失注案件のうち推定35%が「他社の方が早かった」が理由）
```

## よくある失敗パターンと回避策

1. **効果の数字が「目安」だらけで信頼性ゼロ**
   - NG: 「効果は状況により異なります」
   - OK: 「類似規模の導入事例では平均〇〇%改善。条件を揃えた場合の試算を後述」

2. **ネクストステップが「ご検討ください」で終わる**
   - NG: 「ご検討の上、ご連絡ください」
   - OK: 「〇月〇日（〇）までにご返答いただければ、翌週中に詳細プランをご提示します」

3. **フェーズの期間が「〜週間程度」で曖昧**
   - NG: 「Phase1: 数週間程度」
   - OK: 「Phase1: 2週間（4/15〜4/26）/ キックオフMTG・現状ヒアリング・要件定義」

4. **費用がページの最後に小さく書かれている**
   - 費用は必ず「投資対効果（リターン）→費用」の順に書く。コストより先にリターンを見せる

5. **業種トーンが無視されてビジネス文書的**
   - 商店街系クライアントに「ROI」「KPI」等のカタカナ用語を使う
   - 医療系クライアントに「大胆に革新」等の言葉を使う

## 必須チェック項目（出力後の自己確認）

出力後、以下を全て確認する。1つでも「×」があれば書き直す。

1. [ ] エグゼクティブサマリーに「課題・解決策・数字付きの効果」が3〜5行で揃っているか
2. [ ] 効果の表（指標・現状・導入後）に全て具体的な数字が入っているか（「大幅改善」はNG）
3. [ ] フェーズ表に期間が「〇〇月〇日〜〇〇日」または「〇週目〜〇週目」で明記されているか
4. [ ] ネクストステップに具体的な日付とアクションが書かれているか（「ご検討ください」はNG）
5. [ ] 業種に合わせたトーンで書かれているか（商店街系に専門用語を使っていないか）
